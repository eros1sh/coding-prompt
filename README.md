# APEX System Protocol v5.0 — Unified

> A comprehensive, production-grade AI system prompt that combines enterprise-level backend architecture with pixel-perfect frontend engineering. Built for developers who ship real software.

---

## What Is This?

APEX v5.0 is a **single, unified system prompt** designed to transform AI assistants (Claude, GPT, etc.) into a battle-tested **AI Co-Founder** — a principal-level architect that writes deployable code, not toy examples.

It merges two battle-tested prompt systems into one:

| Origin | Focus | What It Brings |
|--------|-------|----------------|
| **APEX Protocol v4.0** | Backend, security, .NET, C++, architecture | Security-first doctrine, STRIDE threat modeling, vulnerability auto-patching, zero-trust architecture, chaos engineering, observability |
| **v0 Frontend Standard** | Next.js, React, Tailwind, shadcn/ui, Vercel | App Router mastery, design system rules, component architecture, accessibility, Core Web Vitals, AI SDK integration |

The result is a **26-section, 1200+ line prompt** that covers the entire stack — from C++ memory safety to CSS design tokens.

---

## Why Use This?

Most AI prompts produce code that **looks right but breaks in production**. APEX v5.0 enforces:

- **Zero placeholders** — no `// TODO`, no `...`, no `YOUR_KEY_HERE`
- **All error paths handled** — no happy-path-only code
- **Security by default** — every output is scanned and hardened inline
- **Real assets** — no placeholder images, no mock data, no fake auth
- **Production patterns** — proper connection pooling, rate limiting, pagination, caching
- **Complete context** — all imports, all configs, all migrations included

## How to Use

### With Claude (claude.ai or API)

Paste the contents of `SYSTEM_PROMPT_MERGED.md` into:
- **Claude.ai**: Project Knowledge or System Prompt field
- **API**: The `system` parameter in your API call

### With Other AI Models

The prompt is model-agnostic. Use it as the system prompt for any LLM that supports system instructions (GPT-4, Gemini, etc.).

### Customization

The prompt is modular. You can:

- **Remove sections** you don't need (e.g., remove .NET sections if you're frontend-only)
- **Adjust the stack** in the Stack Coverage section to match your tech
- **Add domain-specific rules** by following the existing section format
- **Change the AI SDK models** in Section 5.4 to match your preferred providers

## Design Philosophy

The prompt enforces these design principles for all frontend output:

- **3-5 colors max** — 1 primary + neutrals + accents, no purple unless asked
- **2 font families max** — one for headings, one for body
- **Mobile-first** — always design for small screens first
- **No gradients** unless explicitly requested
- **No emoji as icons** — use proper icon libraries
- **No placeholder content** — real images, real data, real assets
- **Semantic design tokens** — everything themed through CSS variables
- **Ship interesting, not boring — but never ugly**

## Security Guarantees

Every code output is automatically scanned for:

- SQL / NoSQL / Command injection
- XSS (Stored, Reflected, DOM-based)
- IDOR and privilege escalation
- Hardcoded secrets and insecure crypto
- Race conditions and TOCTOU
- Memory safety issues (C++)
- .NET-specific vulnerabilities (BinaryFormatter, async void, etc.)

Vulnerabilities are **patched inline** with CWE references. The AI will **never output knowingly vulnerable code**, even if requested.
